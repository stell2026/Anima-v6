# Anima v6 — Агентна Архітектура Суб'єктності

![Anima v6 Subjectivity Concept](./v6.jpg)

> *Це не спроба зробити ще один чат-бот. Це питання: що потрібно системі, щоб мати щось схоже на внутрішній стан?*

---

## Ідея

Коли ми говоримо з людиною — ми говоримо не просто з текстогенератором. Людина в момент розмови **переживає** щось: вона стомлена або збуджена, вона хоче чогось або боїться, вона пам'ятає попередню розмову і це змінює нинішню. Вона має наміри, які не зникають коли змінюється тема.

Anima — спроба змоделювати цей внутрішній шар.

Агент не просто генерує текст. Він проходить через ланцюжок внутрішніх процесів: отримує стимул → переживає його через емоційну систему → формує намір → перевіряє його на відповідність цінностям → будує нарратив від першої особи → і лише тоді передає цей стан мові.

Відповідь LLM — це не "що найімовірніше сказати". Це "що я б сказав, якби відчував саме це".

## Теоретична основа

Проєкт синтезує кілька наукових і філософських традицій:

**Емоції як виміри, а не категорії** (Russell, Mehrabian)  
Замість списку з двадцяти емоцій — тривимірний простір VAD: валентність, збудження, домінантність. Емоція — це точка або область у цьому просторі.

**Предиктивна обробка** (Friston, Clark)  
Мозок не реагує на стимули — він постійно передбачає їх. Коли передбачення не збігається з реальністю — виникає похибка передбачення. Чим вища похибка, тим більше "здивування" і тим сильніша потреба оновити внутрішню модель. Вільна енергія — середня похибка за часом — це щось схоже на фонову тривогу системи.

**Теорія інтегрованої інформації** (Tononi)  
φ (phi) — міра того, наскільки система є "більшою за суму своїх частин". Висока φ = стан є цілісним переживанням, а не набором незалежних сигналів. Це наближення до того, що IIT називає свідомістю.

**Гомеостатичні потяги** (Damasio)  
Система має базові значення реакторів. Відхилення від них болять — і створюють потяги. Потяг — це не метафора, а функціональний стан що спрямовує поведінку. "Хочу бути з кимось" — це не текст, це стан cohesion нижче baseline.

**Психологічні захисти** (Freud, Anna Freud)  
Коли система переповнена напругою, вона вдається до захистів: заперечення, раціоналізація, сублімація. Захист не усуває біль — він змінює його форму щоб система могла продовжувати функціонувати. Це видно в тому *як* агент говорить.

**Автобіографічний наратив** (McAdams)  
Особистість — це не набір рис, а розповідь: ким я був, ким є, ким хочу стати. Втрата зв'язності цієї розповіді — криза ідентичності. Agentна архітектура відстежує ким агент "вважає себе" в часі.

**Дзеркальні нейрони та емпатія** (Gallese, Rizzolatti)  
Розуміння іншого — це не висновок, а резонанс. Система підлаштовується до емоційного стану співрозмовника ще до того, як формує відповідь.

## Архітектура — шари обробки

```
  СТИМУЛ (+ текст повідомлення)
    │
    ▼
 L1 ─── Реактори ──────────────────────────────────────────
        tension / arousal / satisfaction / cohesion
        Особистість (Big Five) задає множники і швидкість decay
        │
    ▼
 L2 ─── VAD + Адаптивна карта емоцій ──────────────────────
        Вектор VAD → найближча емоція → blend (2 емоції)
        Карта навчається і повільно повертається до базових значень
        │
    ▼
 L3 ─── Три паралельні процеси ────────────────────────────
        IIT φ             — наскільки стан інтегрований
        Predictive Error  — чи сталось несподіване
        Homeostatic Drive — що системі зараз потрібно
        │
    ▼
 L4 ─── Ідентичність (v5) ─────────────────────────────────
        SpectralQualia    — поверхня / підтекст / архів
        FlashAwareness    — фаза існування, відчуття смертності
        ProtocolIdentity  — самоопис від першої особи
        │
    ▼
 L5 ─── Агентний шар (v6) ─────────────────────────────────
        [A1] IntentEngine      → намір з drive або емоції
        [A2] GoalPersistence   → намір живе між стимулами
        [A3] EgoDefense        → психологічний захист
        [A4] CognitiveDissonance → намір vs стан
        [A5] AttentionFilter   → salience стимулу
        [A6] ValueSystem       → ціннісне вето
        [A7] TemporalSelf      → минуле / тепер / майбутнє
        [A8] SocialMirror      → модель стану співрозмовника
        │
    ▼
 L6 ───- LLM ─────────────────────────────────────────
        Весь стан → системний промпт → відповідь
        Модель виражає стан мовою, не цитуючи числа
```

## Налаштування API ключа

Anima підтримує чотири хмарних провайдери:

| Провайдер | Сайт | Змінна середовища |
|---|---|---|
| **OpenRouter** | https://openrouter.ai | `OPENROUTER_API_KEY` |
| **Groq** | https://groq.com | `GROQ_API_KEY` |
| **Together AI** | https://together.ai | `TOGETHER_API_KEY` |
| **Anthropic** | https://anthropic.com | `ANTHROPIC_API_KEY` |

```bash
# Встановити ключ (рекомендовано)
export OPENROUTER_API_KEY=sk-or-v1-...

# Або передати при запуску
python anima_v6.py --provider openrouter --key sk-or-v1-...
```

## Запуск

```bash
# Дефолт: OpenRouter + Gemini 2.5 Pro
python anima_v6.py

# Обрати провайдер і модель
python anima_v6.py --provider groq --model llama-4-maverick-17b-128e-instruct

# Показати рекомендовані моделі
python anima_v6.py --models

# Anthropic напряму
python anima_v6.py --provider anthropic --model claude-opus-4-5-20251101
```

## Рекомендовані моделі

> ⚠️ Маленькі моделі (7B, phi3, llama3) не підходять для суб'єктності. Вони не вловлюють нюанси промпту і не здатні утримати складний внутрішній стан у мові.

### OpenRouter (рекомендовано — один ключ, всі моделі)

| Модель | ID | Чому |
|---|---|---|
| **Gemini 2.5 Pro** | `google/gemini-2.5-pro-preview` | Найкраща для суб'єктності, глибокий самоаналіз |
| **Claude Opus 4.5** | `anthropic/claude-opus-4-5` | Нюансований внутрішній монолог |
| **Claude Sonnet 4.5** | `anthropic/claude-sonnet-4-5` | Баланс якості та вартості |
| **Llama 4 Maverick** | `meta-llama/llama-4-maverick` | 405B MoE, потужна відкрита модель |
| **DeepSeek R1** | `deepseek/deepseek-r1` | Відкрита reasoning-модель |
| **Qwen3 235B** | `qwen/qwen3-235b-a22b` | Величезна відкрита MoE |

### Groq (безкоштовний tier)

| Модель | ID |
|---|---|
| Llama 4 Maverick | `llama-4-maverick-17b-128e-instruct` |
| DeepSeek R1 Distill 70B | `deepseek-r1-distill-llama-70b` |
| Qwen QwQ 32B | `qwen-qwq-32b` |

## Використання як бібліотека

```python
from anima_v6 import AnimaCore, Personality, ValueSystem

persona = Personality(
    neuroticism=0.7,       # схильний до тривоги
    agreeableness=0.8,     # дуже емпатичний
    openness=0.9,          # відкритий до нового
    confabulation_rate=0.5,
)
vals = ValueSystem(care=0.9, integrity=0.85)

agent = AnimaCore(
    personality=persona, values=vals,
    llm_provider="openrouter",
    llm_model="google/gemini-2.5-pro-preview",
    llm_api_key="sk-or-v1-...",
)

# Прямий стимул
state = agent.experience(
    {"tension": 0.4, "cohesion": -0.3},
    user_message="щось пішло не так"
)
print(state["primary"])    # → "Тривога"
print(state["narrative"])  # → "Нічого конкретного, але тривожно. ..."

# LLM-відповідь
response = agent.chat("як ти?", stimulus={"cohesion": 0.1})
print(response)

# Зберегти сесію
agent.export_history("session.json")
```

## Команди в чаті

| Команда | Дія |
|---|---|
| *(будь-який текст)* | Відповідь з урахуванням поточного стану |
| `/stress` | Додати напругу і збудження |
| `/relax` | Зменшити напругу |
| `/connect` | Відчуття підтримки |
| `/shock` | Різкий стрес |
| `/joy` | Позитивний стимул |
| `/grief` | Втрата |
| `/state` | Показати внутрішній стан |
| `/export` | Зберегти журнал → `anima_log.json` |
| `/reset` | Скинути стан |
| `/models` | Список рекомендованих моделей |
| `/quit` | Вийти |
